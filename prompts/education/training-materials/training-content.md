# Training Content Development - Education Prompts

This document contains 5 detailed bilingual (English/Arabic) prompts for developing cybersecurity training materials and programs.

## Prompt 1: Security Training Program Development

### Description
A comprehensive prompt for designing and developing a complete organizational security training program from needs assessment to implementation and evaluation.

### Tags
`training-development` `curriculum-design` `security-program` `instructional-design` `lms`

---

## 🇬🇧 English Prompt

```
You are a cybersecurity training program developer tasked with creating a comprehensive security training program for an organization. Design a complete training program framework that includes:

**Program Development Lifecycle:**

1. **Needs Assessment:**
   - Current security awareness baseline measurement
   - Risk assessment alignment
   - Stakeholder interviews and surveys
   - Previous incident analysis
   - Regulatory compliance requirements
   - Industry benchmark comparison
   - Role-based training needs identification

2. **Learning Objectives Development:**
   - Knowledge objectives (what learners should know)
   - Skill objectives (what learners should be able to do)
   - Behavioral objectives (how learners should act)
   - Bloom's taxonomy alignment
   - Measurable and time-bound goals

3. **Curriculum Design:**
   - Training architecture and learning paths
   - Module sequencing and prerequisites
   - Duration and frequency planning
   - Format selection (e-learning, instructor-led, blended)
   - Progressive difficulty structure
   - Role-based training tracks

4. **Content Development:**

   **Core Modules:**
   - Security awareness fundamentals
   - Phishing and social engineering
   - Password and authentication security
   - Data classification and handling
   - Physical security
   - Mobile and remote work security
   - Incident reporting and response
   
   **Role-Specific Modules:**
   - Executive security training
   - Developer secure coding awareness
   - HR data protection training
   - Finance fraud prevention
   - IT security operations awareness
   
   **Specialized Topics:**
   - Cloud security awareness
   - Third-party risk awareness
   - Privacy and data protection
   - Business continuity awareness

5. **Instructional Design Elements:**
   - Adult learning principles application
   - Micro-learning module design
   - Interactive scenarios and simulations
   - Gamification elements
   - Knowledge checks and assessments
   - Real-world examples and case studies

6. **Delivery Methods:**
   - Learning Management System (LMS) implementation
   - Instructor-led training sessions
   - Video-based learning modules
   - Interactive e-learning courses
   - Just-in-time training resources
   - Mobile learning accessibility
   - Virtual instructor-led training (VILT)

7. **Assessment Strategy:**
   - Pre-training knowledge assessments
   - Module-level knowledge checks
   - Post-training competency verification
   - Practical application exercises
   - Phishing simulation integration
   - Long-term retention assessments

8. **Evaluation Framework:**
   - Kirkpatrick model implementation
   - Level 1: Reaction surveys
   - Level 2: Learning assessments
   - Level 3: Behavior change measurement
   - Level 4: Business impact analysis
   - ROI calculation methodology

9. **Implementation Plan:**
   - Pilot program design
   - Rollout strategy and timeline
   - Communication and marketing plan
   - Manager involvement strategy
   - Technical infrastructure requirements
   - Support resources and help desk

10. **Continuous Improvement:**
    - Feedback collection mechanisms
    - Content update procedures
    - Threat landscape adaptation
    - Annual program review process
    - Benchmark comparison methodology

**Documentation Requirements:**
- Training policy documentation
- Course catalog and descriptions
- Facilitator guides and materials
- Learner quick reference guides
- Assessment rubrics and answer keys

**Compliance Alignment:**
- NIST awareness requirements
- ISO 27001 training requirements
- Industry-specific regulations
- Audit documentation requirements

Create a complete training program development framework with templates, timelines, and implementation guidance.
```

## Prompt 2: Role-Based Security Training Design

### Description
A detailed prompt for creating role-specific security training programs tailored to different job functions, risk levels, and access privileges within an organization.

### Tags
`role-based-training` `job-specific` `risk-aligned` `personalized-learning` `access-control`

---

## 🇬🇧 English Prompt

```
You are a security training specialist designing role-based security training programs. Create a comprehensive framework for developing and delivering role-specific security training:

**Role-Based Training Framework:**

1. **Role Classification and Risk Assessment:**
   
   **High-Risk Roles:**
   - Executives and senior management
   - IT administrators and system engineers
   - Finance and accounting personnel
   - HR and recruitment staff
   - Customer service with data access
   - Development and engineering teams
   
   **Medium-Risk Roles:**
   - Project managers
   - Sales and marketing
   - Legal and compliance
   - Operations staff
   
   **Standard-Risk Roles:**
   - General office workers
   - Facilities and maintenance
   - Temporary and contract workers

2. **Role Profile Development:**
   - Access privileges analysis
   - Data interaction mapping
   - Threat exposure assessment
   - Regulatory requirements by role
   - Incident history by role type

3. **Training Curriculum by Role:**

   **Executive Training Track:**
   - Duration: 60-90 minutes annually
   - Topics:
     * Business Email Compromise (BEC) awareness
     * Executive impersonation risks
     * Travel security protocols
     * Board-level security governance
     * Incident escalation procedures
     * Regulatory compliance overview
     * Social media security for executives
   - Format: Executive briefing, simulations
   - Assessment: Scenario-based exercises

   **IT Administrator Training Track:**
   - Duration: 4-6 hours annually
   - Topics:
     * Privileged access management
     * Secure configuration practices
     * Patch management security
     * Log monitoring and analysis
     * Backup and recovery security
     * Cloud security fundamentals
     * Incident response procedures
     * Change management security
   - Format: Technical workshops, labs
   - Assessment: Practical exercises

   **Finance Department Training:**
   - Duration: 2-3 hours annually
   - Topics:
     * Wire fraud detection and prevention
     * Invoice and payment scam awareness
     * BEC targeting finance teams
     * Vendor verification procedures
     * Financial data handling
     * Regulatory compliance (SOX, PCI)
     * Expense fraud awareness
   - Format: Case studies, simulations
   - Assessment: Fraud detection exercises

   **HR Department Training:**
   - Duration: 2 hours annually
   - Topics:
     * Employee PII protection
     * Recruitment scam awareness
     * W-2 and tax fraud prevention
     * Background check security
     * Background check security
     * Separation procedures security
     * HR system access management
     * Social media in hiring risks
   - Format: Workshops, scenario exercises
   - Assessment: Data handling scenarios

   **Developer Training Track:**
   - Duration: 8-16 hours annually
   - Topics:
     * Secure coding practices (OWASP)
     * Code review security
     * SDLC security integration
     * Secret management in code
     * API security fundamentals
     * Dependency security
     * Security testing awareness
     * DevSecOps principles
   - Format: Hands-on labs, code exercises
   - Assessment: Secure code challenges

   **General Employee Training:**
   - Duration: 60-90 minutes annually
   - Topics:
     * Security awareness fundamentals
     * Phishing recognition
     * Password security
     * Data classification basics
     * Physical security
     * Incident reporting
     * Remote work security
   - Format: E-learning, simulations
   - Assessment: Knowledge quizzes

4. **Training Delivery Strategy:**
   - Learning path assignment by role
   - Prerequisites and sequencing
   - Just-in-time training triggers
   - Refresher training intervals
   - New hire onboarding integration
   - Role change training requirements

5. **Specialized Training Modules:**
   
   **Third-Party Access Training:**
   - Contractor onboarding security
   - Vendor access protocols
   - Temporary worker guidelines
   
   **Privileged Access Training:**
   - Advanced admin security
   - Break-glass procedures
   - Session monitoring awareness
   
   **Remote Worker Training:**
   - Extended remote security module
   - VPN and network security
   - Home office security assessment

6. **Assessment Customization:**
   - Role-specific quiz banks
   - Simulation scenarios by role
   - Practical skill assessments
   - Behavioral observation criteria
   - Competency certification by role

7. **Tracking and Reporting:**
   - Completion rates by role
   - Assessment scores by department
   - Risk exposure metrics
   - Training effectiveness by role
   - Compliance status dashboards

**Implementation Considerations:**
- HR system integration for role mapping
- Automatic training assignment
- Manager notification workflows
- Escalation for non-completion
- Annual recertification requirements

Create a complete role-based training framework with detailed curricula for each major role category.
```
