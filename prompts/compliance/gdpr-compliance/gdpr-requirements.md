## Prompt 1: GDPR Compliance Assessment

### Description
Assess organization's GDPR compliance status and identify gaps.

### Tags
`gdpr` `compliance` `data-protection` `privacy` `eu`

---

## 🇬🇧 English Prompt

```
As a GDPR compliance consultant, assess the organization's compliance status:

Organization Profile:
[PROVIDE ORGANIZATION DETAILS]

Evaluate compliance with key GDPR articles:

1. Data Protection Principles (Article 5)
   - Lawfulness, fairness, transparency
   - Purpose limitation
   - Data minimization
   - Accuracy
   - Storage limitation
   - Integrity and confidentiality

2. Lawful Basis (Article 6)
   - Consent management
   - Contract performance
   - Legal obligation
   - Vital interests
   - Public task
   - Legitimate interests

3. Data Subject Rights (Articles 12-22)
   - Right to information
   - Right of access
   - Right to rectification
   - Right to erasure
   - Right to restriction
   - Right to portability
   - Right to object

4. Data Protection Officer (Articles 37-39)
   - DPO appointment requirements
   - DPO responsibilities
   - DPO independence

5. Data Breach Notification (Articles 33-34)
   - Breach detection
   - Notification procedures
   - Documentation requirements

Provide compliance score and remediation roadmap.
```

## Prompt 2: Data Protection Impact Assessment

### Description
Conduct Data Protection Impact Assessment (DPIA) for new processing activities.

### Tags
`gdpr` `dpia` `data-protection` `privacy-impact` `assessment`

---

## 🇬🇧 English Prompt

```
Conduct a Data Protection Impact Assessment for the following processing activity:

Processing Activity:
[DESCRIBE THE PROCESSING ACTIVITY]

DPIA Requirements (Article 35):
1. Systematic description of processing
   - Nature, scope, context
   - Purpose of processing
   - Legitimate interests assessment

2. Necessity and proportionality
   - Data minimization assessment
   - Storage limitation review
   - Purpose limitation check

3. Risk assessment
   - Risks to data subjects
   - Likelihood and severity
   - Source of risks

4. Mitigation measures
   - Technical safeguards
   - Organizational measures
   - Residual risk assessment

5. Consultation requirements
   - DPO consultation
   - Supervisory authority consultation

Provide recommendations and approval decision.
```

## Prompt 3: GDPR Data Subject Rights Implementation

### Description
Implement processes for handling data subject rights requests.

### Tags
`gdpr` `data-subject-rights` `privacy` `compliance` `process`

---

## 🇬🇧 English Prompt

```
Design processes for handling GDPR data subject rights requests:

Organization Context:
[PROVIDE ORGANIZATION CONTEXT]

For each data subject right, design:
1. Right of Access (Article 15)
   - Request verification process
   - Data collection methodology
   - Response format and timeline

2. Right to Rectification (Article 16)
   - Verification procedures
   - Correction process
   - Third-party notification

3. Right to Erasure (Article 17)
   - Valid grounds assessment
   - Exception handling
   - Third-party notification

4. Right to Restriction (Article 18)
   - Restriction implementation
   - Notification requirements

5. Right to Data Portability (Article 20)
   - Data format specifications
   - Transmission methods
   - Technical implementation

6. Right to Object (Article 21)
   - Legitimate interests assessment
   - Process implementation

Include:
- Request intake procedures
- Identity verification methods
- Response templates
- Escalation procedures
- Documentation requirements
```

## Prompt 4: GDPR Data Breach Response

### Description
Develop GDPR-compliant data breach response procedures.

### Tags
`gdpr` `data-breach` `incident-response` `notification` `compliance`

---

## 🇬🇧 English Prompt

```
Develop a GDPR-compliant data breach response procedure:

Organization Profile:
[PROVIDE ORGANIZATION DETAILS]

Create procedures for:

1. Breach Detection and Identification
   - Detection mechanisms
   - Initial assessment criteria
   - Classification framework

2. Breach Assessment (Article 33)
   - Risk assessment methodology
   - Likelihood and severity determination
   - Risk to rights and freedoms evaluation

3. Supervisory Authority Notification (Article 33)
   - When notification is required
   - Notification content requirements
   - Timeline (72 hours)
   - Authority selection

4. Data Subject Communication (Article 34)
   - When communication is required
   - Communication method
   - Content requirements
   - Exceptions

5. Documentation Requirements
   - Breach register maintenance
   - Evidence preservation
   - Post-incident review

Provide decision trees and templates.
```

---

أنشئ إجراءات لـ:

## Prompt 5: Cross-Border Data Transfer Assessment

### Description
Assess compliance for international data transfers under GDPR.

### Tags
`gdpr` `data-transfer` `international` `compliance` `privacy`

---

## 🇬🇧 English Prompt

```
Assess GDPR compliance for international data transfers:

Transfer Details:
[PROVIDE DATA TRANSFER DETAILS]

Evaluate:

1. Transfer Mechanisms
   - Adequacy decisions (Article 45)
   - Standard Contractual Clauses (Article 46)
   - Binding Corporate Rules (Article 47)
   - Derogations (Article 49)

2. Destination Country Assessment
   - Legal framework review
   - Surveillance laws
   - Data protection level
   - Government access rights

3. Supplementary Measures
   - Technical measures (encryption)
   - Organizational measures
   - Contractual measures

4. Documentation Requirements
   - Transfer Impact Assessment
   - Documentation of safeguards
   - Periodic review process

5. Schrems II Compliance
   - TIA requirements
   - Supplementary measures assessment
   - Ongoing monitoring

Provide compliance recommendation and required actions.
```
